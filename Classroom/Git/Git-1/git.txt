극단적으로 아주 많은 수의 코드가 있다고 가정할 때,
코드 단 한 개의 값을 바꾼다면 바뀐 부분을 감지하기 어려울 것이다.

'변경된 부분'만 기록하는 Git은 혁신적인 도구로 개발자들 사이에서는 완성도와 검증이 완료되어 표준화 되었다.

GUI 방식으로도 조작이 가능하지만, 약간은 불편하더라도 CLI 방식이 지원하는 정도와 커스텀 하는 측면에서 압도적으로 강력함.
또, GUI 방식은 너무도 간편해서 언제든 사용이 가능한데 반해, CLI 방식은 훈련이 필요함.
따라서, CLI 방식으로 VS Code의 터미널에서 작업하는 것을 손에 익힐 예정임.

# 주요 명령어
1. git init
2. git config
3. git add [파일명]
4. git commit -m "[커밋 내용]"
5. git status
6. git log
7. git diff

가볍게 다른 명령어도 검색해보고, Git을 다뤄보세요.
Git에 국한되지 않는 CLI 명령 개념을 접하는 데에 좋은 수단이 될 것.
